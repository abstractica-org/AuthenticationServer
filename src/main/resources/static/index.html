OAuth2 Authorization Server with JWT Token Support

Overview

Welcome to the Authentication Server! This is a secure OAuth2 authorization server that handles user authentication, registration, and token management for your applications.

  • User registration and email verification
  • OAuth2 and JWT token-based authentication
  • Two-factor authentication (2FA) support
  • Password reset functionality
  • User profile management
  • Client credentials management
  • Comprehensive audit logging

Quick Start Guide

1. Register a New User

POST /api/auth/register

Create a new user account:

{ "username": "user@example.com", "password": "SecurePassword123!", "email": "user@example.com" }

2. Verify Email

POST /api/auth/verify-email

Verify the user's email address using the token sent to their email:

{ "token": "verification-token-from-email" }

3. Authenticate (Login)

POST /api/auth/login

Authenticate with username and password:

{ "username": "user@example.com", "password": "SecurePassword123!" }

Response includes: access_token, refresh_token, and token_type

üí° Tip: Use the access_token in the Authorization header: Authorization: Bearer {access_token}

Integrating with a Client Application

Step-by-Step Integration Guide

  1. User Registration: POST to /api/auth/register with user credentials
  2. Email Verification: User clicks link in email and POST to /api/auth/verify-email
  3. User Login: POST to /api/auth/login to obtain access token
  4. API Requests: Include token in Authorization header: Bearer {access_token}
  5. Token Refresh: Use refresh_token to obtain new access_token when expired
  6. Logout: POST to /api/auth/logout to invalidate tokens

Sample JavaScript Fetch Implementation

// Register a user const registerUser = async (email, password) => { const response = await fetch('/api/auth/register', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ username: email, email, password }) }); return response.json(); }; // Login const loginUser = async (email, password) => { const response = await fetch('/api/auth/login', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ username: email, password }) }); const data = await response.json(); localStorage.setItem('accessToken', data.access_token); return data; }; // Make authenticated request const fetchUserProfile = async () => { const token = localStorage.getItem('accessToken'); const response = await fetch('/api/users/me', { headers: { 'Authorization': `Bearer ${token}` } }); return response.json(); };

Sample cURL Examples

# Register a user curl -X POST http://localhost:8080/api/auth/register \ -H "Content-Type: application/json" \ -d '{"username":"user@example.com","email":"user@example.com","password":"SecurePassword123!"}' # Login curl -X POST http://localhost:8080/api/auth/login \ -H "Content-Type: application/json" \ -d '{"username":"user@example.com","password":"SecurePassword123!"}' # Get user profile (requires token) curl -X GET http://localhost:8080/api/users/me \ -H "Authorization: Bearer YOUR_ACCESS_TOKEN"

Security Features

  • Passwords are securely hashed using bcrypt
  • JWT tokens with configurable expiration
  • CORS protection enabled
  • HTTPS recommended for production
  • Email verification required for accounts
  • Two-factor authentication (2FA) support
  • Audit logging for all authentication events
  • Rate limiting on authentication endpoints
